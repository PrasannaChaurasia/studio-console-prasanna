This report translates the current reference set into an original direction for Studio Console - Prasanna. It is not a clone brief. It identifies reusable UX patterns, motion logic, navigation structure, and portfolio architecture that can be adapted ethically.

## Design Principles For This Portfolio

1. The portfolio should behave like a workstation: visitors can inspect, filter, preview, and open source PDFs.
2. The home page should quickly answer: who is Prasanna, what kind of work exists, what is the strongest case study, and how do I contact him?
3. The profile page should behave like a professional dossier, not a generic About page.
4. Project pages should become case-study rooms: image/preview, concept, process, tools, role, output, and downloadable proof.
5. Motion should clarify hierarchy: page reveal, hover inspection, sticky tools, and selected states.
6. Navigation should stay simple, but the content below can feel experimental.
7. The visual language should be original: cinematic, architectural, AI/BIM-informed, and not a direct copy of any reference.

## Immediate Website Framework

- Home: landing, reference-led studio manifesto, destination grid, featured project, interactive console, portfolio layers, CTA.
- Work: full index with filterable categories and case-study links.
- Conceptuals: speculative projects and cinematic worlds.
- Real Projects: professional project evidence and role clarity.
- BIM: documentation, tools, ISO 19650 positioning, and Veridian Elan.
- Profile: CV-led professional dossier with live CV preview.
- Archive: source PDFs and downloadable proof.
- Contact: direct communication routes and collaboration framing.
